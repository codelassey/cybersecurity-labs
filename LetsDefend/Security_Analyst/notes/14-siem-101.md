# SIEM 101 - SOC Analyst Notes

## What is SIEM?
**Security Information and Event Management (SIEM)** is a platform that:
- Collects logs from multiple systems
- Stores and normalizes the logs
- Correlates events to detect threats
- Generates alerts for SOC analysts

> **Goal:** Centralize logs and identify suspicious activity in real time.

---

# SIEM Workflow

```text
Devices & Applications
        │
        ▼
 Log Collection
        │
        ▼
 Log Aggregator
 (Parse • Filter • Enrich) DOES NOT ANALYZE
        │
        ▼
   Log Storage
        │
        ▼
 Detection Rules
        │
        ▼
     SIEM Alerts
        │
        ▼
   SOC Investigation
```

# Log Collection

A **log** is a record of an event generated by an operating system, application, or network device.

Common log sources:
- Endpoints (Windows/Linux)
- Firewalls
- IDS/IPS
- Routers & Switches
- Proxy servers
- Web servers
- Cloud services

Without logs, a SIEM cannot detect threats.

---

## Log Collection Methods

### 1. Agent-Based Collection  (Most Common)

A lightweight agent is installed on the host to collect and forward logs.

Examples:
- Splunk Universal Forwarder
- Elastic Beats
- NXLog
- ArcSight Connector

**Advantages**
- Reliable
- Supports parsing & filtering
- Encryption (TLS)
- Buffering during outages
- Log integrity checks

**Disadvantages**
- Uses CPU and RAM
- Requires installation and updates

---

### 2. Agentless Collection

Logs are collected remotely using protocols like:
- SSH
- WMI

**Advantages**
- No software installation
- Easier deployment

**Disadvantages**
- Limited functionality
- Credentials required (security risk)
- Less reliable than agents

---

### 3. Manual Collection

Used when no agent exists.

Example:
- Custom scripts
- Cloud API integrations
- Vendor-specific log collectors

---

# Syslog

A standard protocol for sending logs across the network.

Supports:
- UDP
- TCP
- TLS (encrypted)

Common syslog fields:
- Timestamp
- Source Device
- Facility
- Severity
- Message ID
- Message

Used by:
- Firewalls
- Routers
- Linux servers
- IDS/IPS
- Network devices

---

# Log Aggregation

A **Log Aggregator** receives logs before they reach the SIEM.

Its responsibilities include:

- Collecting logs
- Parsing formats
- Filtering unnecessary data
- Enriching logs
- Forwarding processed logs

---

## Parsing

Converts logs into a standardized format so the SIEM can understand them.

Example:

Before
```
User: John logged in from 10.0.0.5
```

After
```
Username = John
Source_IP = 10.0.0.5
Action = Login
```

---

## Filtering

Reduces unnecessary data.

Example:
- Keep only HTTP status codes
- Ignore noisy events
- Remove duplicate logs

This improves SIEM performance.

---

## Log Enrichment

Adds useful context to logs.

Common enrichments:
- Geolocation (Country, City)
- DNS lookups
- Reverse DNS
- Asset information
- User information

Enrichment helps analysts investigate faster.

---

## Log Modification

Standardizes inconsistent logs.

Examples:
- Convert date formats
- Normalize time zones
- Rename inconsistent fields

This makes searches and correlation more accurate.

---

# EPS (Events Per Second)

**EPS = Total Events ÷ Seconds**

Example:

1000 logs in 5 seconds

```
EPS = 1000 ÷ 5 = 200 EPS
```

Higher EPS means:
- More processing power required
- Larger storage
- Better scalability

---

# Scaling the Aggregator

Large environments use **multiple aggregators** to distribute incoming logs.

Benefits:
- Prevents overload
- Improves performance
- Supports higher EPS

---

# Log Storage

The SIEM stores processed logs for:
- Searching
- Investigations
- Compliance
- Threat hunting

A good SIEM storage solution should provide:

- Fast searches
- High scalability
- Reliable long-term retention

---

## WORM Storage

**Write Once, Read Many**

Characteristics:
- Logs cannot be modified after writing
- Preserves evidence integrity
- Ideal for forensic investigations
- Common in SIEM environments

---

# Key SOC Takeaways

- SIEM is only as effective as the quality of its logs.
- Agent-based collection is generally preferred over agentless.
- Log aggregation improves data quality through parsing, filtering, and enrichment.
- Enriched logs provide additional investigation context.
- EPS measures log ingestion performance.
- Storage speed is just as important as storage capacity.
- WORM storage helps preserve log integrity for investigations.

---

# SIEM Investigation Flow

1. Logs generated by devices
2. Logs collected (Agent / Agentless)
3. Logs aggregated
4. Parse, filter, and enrich logs
5. Store logs
6. Detection rules analyze events
7. SIEM generates alerts
8. SOC analyst investigates and responds
